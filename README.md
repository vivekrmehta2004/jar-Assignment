## What the script does, in order

1. **Load & clean** — reads all three Excel files. `Order Date` is a mix of real dates and text strings, so it gets normalized with `pd.to_datetime(..., dayfirst=True)`. Blank rows are dropped.
2. **Merge** — `Order Details` + `List of Orders` joined on `Order ID` → one table with sales, profit, category, state, city, and date all together (1,500 line items / 500 orders).
3. **Part 1 — Category profitability**: total sales, total profit, avg profit per order, and profit margin per category.
4. **Part 2 — Furniture target vs actual**: the target file's dates needed reconstructing (the day-of-month field actually encodes the short year, e.g. day=18 → 2018 — see `fix_month()` in the script). Month-over-month target change is calculated, then compared against actual monthly Furniture sales to get an achievement %.
5. **Part 3 — Regional performance**: top 5 states by order count, then sales/profit for those states, then a city-level breakdown inside each of those states to spot which specific cities are dragging performance down.

## Known data quirks (worth knowing before you dig further)

- **Sales_target.xlsx dates**: stored oddly — the "day" of the date is really the last 2 digits of the year (format was `mmm-yy` originally). The script corrects for this; double check `fix_month()` still makes sense if you're using a different copy of the file.
- **A few "Delhi" rows appear under Madhya Pradesh** in `List_of_Orders.xlsx`. Likely a data entry error — flagged in the report rather than treated as a real signal.
- **Order dates in `List_of_Orders.xlsx`** span Jan 2018–Dec 2019, but the target file only covers Apr 2018–Mar 2019 (one fiscal year), so Part 2's actual-vs-target comparison is limited to that overlapping window.
